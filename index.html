Ориентирован на клиентов

GraphQL стремится удовлетворить требования разработчиков нативных приложений и фронтендеров.

Продукт-центированный

В приложениях без GraphQL структуру ответа определяет код на стороне сервера. Запросы GraphQL запрашивают ровно столько данных сколько нужно данному клиенту.

Упрощает обратную совместимость

Facebook выкатывает новый клиент каждые 2 недели и поддерживает его как минимум в течение 2х лет. 52 версии приложений одновременно используют один и тот же GraphQL API.

Не привязан к источнику данных

GraphQL можно использовать с любыми источниками данных. SQL, NoSQL, REST, FS, etc...

Не привязан к транспортному уровню

GraphQL - протокол уровня приложения и не привязан к протоколу транспортного уровня. GraphQL запрос - это строка которая разбирается и обрабатывается провайдером данных.

Не привязан к конкретной реализации

Javascript Ruby PHP Python Java C/C++ Go Scala .NET Elixir Haskell SQL Lua

Сильно типизированный

GraphQL - сильно типизированный язык запросов. Это позволяет инструментам разработки валидизировать запрос не полагаясь на проверки этапа выполнения.

Самясьвядомы

Клиент может делать запросы к системе типов GraphQL используя GraphQL. Упрощает разработку клиентов для языков со статической типизацией (Java, Objective-C, Swift). Клиент всегда знает о типах данных ответа и о схеме типов у graphql провайдера.

Relay

Фреймворк для работы с данными в React приложениях :)

Фреймворк для работы с данными в React приложениях

Фреймворк для работы с данными в приложениях с иерархическими видами.

Декларативность

Relay позволяет декларативно описать данные, которые нужны для компонента. Relay сам решает когда и как добыть эти данные.

Соразмещение

Запросы размещены в компонетах. Relay аггрегирует запросы из компонетов чтобы запросить у сервера только те данные, которые действительно нужны.

Модификации

Relay позволяет модифицировать данные на клиенте и на сервере с помощью GraphQL mutations.
